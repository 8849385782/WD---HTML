Anchor Tags & Image Tags

The <a href="..."> tag creates a hyperlink. Anything between the opening and closing tag becomes clickable. Adding target="_blank" makes the link open in a new browser tab, and pairing it with rel="noopener noreferrer" keeps the new tab from being able to access the original page for security reasons.

<a href="https://www.instagram.com/example/" target="_blank" rel="noopener noreferrer">
  Follow on Instagram
</a>

The <img src="..."> tag embeds an image. It is a self-closing (void) element — it never wraps content, it just points to a file.

<img src="images/photo.jpg" alt="Description of the photo">

Relative vs. Absolute Paths

  • Relative path — points to a file location relative to the current HTML file, e.g. images/artist.jpg or ../assets/photo.png. Useful when the image lives inside your own project folder, since the link keeps working if you move the whole folder or host it somewhere else.
  • Absolute path — the full web address of a file, e.g. https://example.com/images/artist.jpg. Needed when the image is hosted on a different website, but it breaks if that external site ever removes or moves the file.

Alt Attributes and SEO

The alt attribute is a text description of an image. It matters for two main reasons:

  • Accessibility — screen readers speak the alt text aloud for visually impaired visitors, so it should describe exactly what's in the image and why it matters, not just repeat a filename.
  • SEO — search engines can't "see" pictures, so they read alt text to understand what an image shows, which helps the page rank for related searches. Good alt text is specific — naming the subject, e.g. "Arijit Singh photo" — rather than vague, e.g. "image1".
