🔍 Front-end vs Server-side Validation

Front-end validation (using HTML attributes like required and pattern) runs instantly in the user's browser, before anything is sent anywhere — it's fast and gives immediate feedback, but it only checks what the browser is told to check.

Server-side validation happens after the form is submitted, on a server the user doesn't control. It re-checks every value against the real rules (format, length, duplicates, business logic) because the server can never fully trust data coming from a browser.

Risk example: if a login form only relies on front-end required/pattern checks, someone could bypass the browser entirely — using a tool like curl or Postman to send a raw POST request straight to the server — and submit empty fields or malicious input (like a SQL injection string) that the HTML page would normally have blocked. Without server-side validation, that bad data goes straight into the backend unchecked.
